# AGENTS.md — Subagent Coordination Protocol

This document defines the conventions subagents (Forge, Scholar, Ralph, etc.)
must follow when handing off results to the main agent (Maeve).

---

## Structured Handoff Blocks (Issue #615)

All subagents **MUST** emit a structured YAML handoff block upon task completion.
Unstructured prose summaries degrade across context resets and lose critical details.

### Schema

```yaml
handoff:
  task_id: <string>            # Short unique ID, e.g. "forge-pr583-fix"
  goal: <string>               # One-sentence task description
  status: <completed|failed|partial|escalate>
  completed:                   # What was done (empty list = [])
    - "<action at file:line>"
  pending:                     # Work left open
    - "<item>"
  risks:                       # Concerns to surface to main agent
    - "<risk>"
  artifacts:                   # Concrete outputs produced
    - type: <commit|pr|file|branch|test-result|report|issue|other>
      ref: <SHA|PR#|path>
      branch: <optional>
      description: <optional>
  verification:                # Checks run and their outcomes
    - "npm test: N tests passed"
    - "tsc --noEmit: clean"
  rollback: "<optional git command or instructions>"
```

### Status Semantics

| Status      | Meaning |
|-------------|---------|
| `completed` | Task finished; all acceptance criteria met. |
| `failed`    | Task could not be completed; escalate immediately. |
| `partial`   | Some work done; `pending[]` lists what remains. |
| `escalate`  | Blocked; needs main-agent decision before continuing. |

### Rules

1. **Always emit a handoff block** — even for failed tasks.
2. **`completed[]` must be non-empty** when `status: completed`.
3. **`artifacts[]` must list every commit, PR, or file touched.**
4. **`verification[]` must include test and lint results.**
5. **`rollback` is required** whenever artifacts include commits or PRs.
6. List **specific file + line** references in `completed[]` items, not vague descriptions.

### Example

```yaml
handoff:
  task_id: forge-pr583-fix
  goal: "Fix ambientBudgetTokens cap in stage-injection.ts"
  status: completed
  completed:
    - "Changed Math.max(100, indexBudget) → Math.max(0, indexBudget) in stage-injection.ts:165"
    - "11 review threads resolved via GraphQL"
  pending: []
  risks: []
  artifacts:
    - type: commit
      ref: e529d11
      branch: feat/581-architecture-refactor-hybrid-memory-core
    - type: pr
      ref: "583"
  verification:
    - "npm test: 3610 tests passed"
    - "tsc --noEmit: clean"
  rollback: "git revert e529d11"
```

## ACTIVE-TASK.md Protocol

In-progress tasks tracked in `ACTIVE-TASK.md` must include a `Handoff:` field
once a subagent has completed its first checkpoint:

```markdown
### [forge-99]: Implement feature X
- **Branch:** feat/99-feature-x
- **Status:** In progress
- **Subagent:** forge-subagent-abc
- **Handoff:** [COMPLETED] Goal | Completed: Step 1 (+2 more) | Pending: 1 item(s)
- **Next:** Review and merge
- **Started:** 2026-03-18T10:00:00.000Z
- **Updated:** 2026-03-18T14:30:00.000Z
```

The `Handoff:` field is generated by `formatHandoffSummary()` in
`extensions/memory-hybrid/services/handoff.ts`.
